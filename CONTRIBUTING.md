# Contributing Guidelines

Thank you for considering contributing to this project! This document outlines the process for contributing to this repository.

## How to Contribute
1. Fork the repository
2. Create a new branch for your feature
3. Make your changes
4. Submit a pull request

## Code Style
- Follow the established coding standards
- Write clear commit messages
- Include Cucumber tests for new features
- Follow DDD architecture principles
- Respect ESLint boundary rules

## Testing Requirements
- Write Cucumber scenarios for new features
- Ensure all tests pass before submitting PRs
- Use behavior-driven development (BDD) approach
- Test both happy path and edge cases

## Pull Request Process
1. Ensure your code follows the style guidelines
2. Run `nx affected:lint` and `nx affected:test`
3. Update documentation as needed
4. Add Cucumber tests for new functionality
5. Submit your pull request with a clear description
